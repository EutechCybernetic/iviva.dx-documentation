# Notification

The Notification section lets users set up and manage alerts that trigger during specific workflow transitions. These alerts keep stakeholders informed about important changes, updates, or actions.

This section helps users and teams communicate quickly when important workflow milestones occur.

**Component 1 → Configure Notification Template**\
Click the **Configure Notification Template** link to open the configuration panel. From there, you can define a new notification rule by specifying when the notification should trigger and who should receive it.

<figure><img src="../../.gitbook/assets/1 (44) (1).png" alt=""><figcaption></figcaption></figure>

**1 → Enable Notification**\
The **Enable Notification** toggle is off by default. Switch it on or off to activate or deactivate the notification.

**2 → Name**\
Enter a unique name for the notification template for easy identification.

**3 → Subject**\
Set the subject line for the notification message.

**4 → Enable SMS**\
Switch on **Enable SMS** to send the notification as a text message.

**5 → Enable Email**

Switch on **Enable Email** to send the notification as an email.

**6 → Recipients (To, CC, BCC)**\
Specify who should receive the notification. Add users to the **To**, **CC**, or **BCC** fields.

Click **+ Add Recipients** to add recipients.

<figure><img src="../../.gitbook/assets/image (53).png" alt=""><figcaption></figcaption></figure>

**1 → Recipient Selection Panel**

Select recipients from the **Recipients, User Group, Email, Mobile,** or **Member** tabs, depending on the fields configured in the form.

**7 → Email Message Tab**\
Allows users to configure the body of the email message to be sent.

**8 → Short Message (SMS) Tab**\
Configure the content of the SMS message, if enabled.

**9 → Push Notification Tab**\
Click the **Push Notification** tab to create the message body for push notifications, delivering real-time alerts within the application.

**10 → Trigger Settings**\
Choose the system event that will trigger the notification. Options include:

* When Transaction Created
* When Transaction’s Details Updated
* When Transaction Deleted
* When Transition Happened
* When Transaction’s Stage Changed To
* When SLA Expired

**Component 2 → Delete Notification**\
Permanently remove the selected notification template from the application.

**Component 3 → Edit Notification**\
Open the editing panel to update notification settings such as recipients, message content, or trigger conditions.

**Component 4 → Duplicate Notification**\
Create a copy of the selected notification rule. Use this to quickly replicate and adjust notifications with similar setups.

**Component 5 → Recipient Count**\
Display the number of recipients for the notification. Click to see detailed recipient information.

**Component 6 → Notification Status**\
Show whether the notification is currently enabled or disabled. Enabled notifications trigger automatically when their conditions are met.

***

The Notification section helps administrators maintain smooth communication during task progression. By setting clear triggers and other notification settings, admins ensure all users stay informed and respond to updates in real time.”
