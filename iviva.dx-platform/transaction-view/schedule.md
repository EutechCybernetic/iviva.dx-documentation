# Schedule

The **Schedule** section provides detailed timing for your task or event. It includes the **Target Start Date and Time**, which marks when the task is planned to begin, and the **Target End Date and Time**, indicates when it is expected to finish.&#x20;

The **Recurrence Pattern** shows how often the task repeats (e.g., weekly, monthly, etc.), and the **Sequence** indicates the order of occurrences (e.g., first, second, etc.).&#x20;

The **Scheduled Start and End Date & Time** reflect the actual scheduled dates and times for each occurrence, helping you stay on track with your planned timeline.

<figure><img src="../../.gitbook/assets/1 (32).png" alt=""><figcaption></figcaption></figure>



* **Component 1:** Clicking on **Schedule** in the sidebar scrolls directly to the Schedule section of the page, allowing easy and direct access.
* **Component 2:** Displays the name of the scheduler used to configure the schedule details.
* **Component 3:** Indicates when the task or activity is intended to begin.
* **Component 4:** Indicates when the task or activity is intended to be completed.
* **Component 5:** Shows how frequently the task is set to repeat (e.g., daily, weekly, monthly).
* **Component 6:** Specifies the position of the occurrence in the recurring schedule (e.g., 1st, 2nd, etc.).
* **Component 7:** Displays the planned start time for each scheduled occurrence.
* **Component 8:** Displays the planned end time for each scheduled occurrence.
* **Component 9:** If any schedule dates are modified, this button allows users to save and apply the changes.
