# Transaction View

The Transaction view lets you to see everything related to a specific task, including its progress, deadlines, and assigned team members. You can easily track the current status and any relevant updates. It also allows you to link and view related tasks, giving you a broader picture of how each task fits into the larger project or workflow. By [connecting related tasks](links.md), you can streamline collaboration, manage dependencies, and ensure everything stays on track for successful completion.

**Component 1:** **Transition Action Bar** - Allows users to perform actions related to the current stage of the transaction, such as acknowledging or canceling a request.

**Component 2:** **Detail View Header** - Displays key information such as the transaction title and status badge (e.g., priority level and reference ID), providing a quick summary of the work request.

**Component 3:** **SLAs Section** - Shows response timelines, including the remaining time, SLA  trigger time, and expiry. This helps ensure timely action on the transaction.

**Component 4:** **Conversation Area** - Enables users to initiate or view ongoing chats related to the transaction, supporting team collaboration and communication within the same interface.

**Component 5:** **Transaction Details** - Displays all transaction details entered or updated so far, such as inventory store, related members, issue location, and other form fields.

**Component 6:** **Sidebar Navigation** - Provides quick links to various sections like details, related members, checklists, attachments, and transition history, making it easy to move between different parts of the transaction.
